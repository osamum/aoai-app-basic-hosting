# 演習 2-2 : ログの有効化と取得について

アプリケーションやサービスの調査を実施する際、それらの過去の状態を把握するにはログを参照する必要があります。

Azure が提供するサービスのリソースにはログを取得するための豊富な機能が用意されていますが、既定の状態でそれらがすべて有効になっているわけではありません。

それらを有効にし、ログを詳細に分析するには [Log Analytics](https://learn.microsoft.com/ja-jp/azure/azure-monitor/logs/log-analytics-workspace-overview) ワークスペースや [Azure Monitor](https://learn.microsoft.com/ja-jp/azure/azure-monitor/fundamentals/overview) などの他のサービスと連携する必要があります。

これについては [演習 3-3 : 高度なログ監視](Ex03-3.md) で詳しく説明しますが、ここでは App Service が提供しているログの有効化と取得方法と、Azure OpenAI サービスが提供しているメトリックの確認方法について説明します。

> [!NOTE]
この演習 2-2 では App Service 単体でのログの取得方法と Azure OpenAI サービスのメトリックの確認方法についてのみ説明します。Azure Storage や Azure AI Search サービスのログの取得、さらには Azure Monitor が提供するさまざまな監視やログ分析機能を使用する機能については [演習 3-3 : 高度なログ監視](Ex03-3.md) で説明します。

一般的な Web サーバーの場合、既定の状態でアクセスログやエラーログなどのログが出力されますが、App Service では既定ではログが出力されないため、ログの設定と有効化を行う必要があります。
